# git gc

Runs garbage collection to clean up unnecessary files and optimize the local repository.

## Basic Usage

```shell
# Run garbage collection
$ git gc
Enumerating objects: 1250, done.
Counting objects: 100% (1250/1250), done.
Delta compression using up to 8 threads
Compressing objects: 100% (450/450), done.
Writing objects: 100% (1250/1250), done.
Total 1250 (delta 800), reused 1100 (delta 700)
```

## What gc Does

```shell
# git gc combines several housekeeping tasks:
# 1. Packs loose objects into packfiles (git repack)
# 2. Removes unreachable objects older than gc.pruneExpire (git prune)
# 3. Packs refs (git pack-refs)
# 4. Removes old reflog entries (git reflog expire)
# 5. Prunes worktree metadata (git worktree prune)
# 6. Re-computes commit-graph (if enabled)

# See what gc would run
$ git gc --dry-run
```

## Aggressive GC

```shell
# More thorough (but slower) — better compression
$ git gc --aggressive
Enumerating objects: 1250, done.
Counting objects: 100% (1250/1250), done.
Delta compression using up to 8 threads
Compressing objects: 100% (1250/1250), done.
Writing objects: 100% (1250/1250), done.
Total 1250 (delta 950), reused 0 (delta 0)

# Aggressive gc uses more CPU for better delta compression
# Only run occasionally (e.g., after importing a large history)
```

## Auto GC

```shell
# Git runs gc automatically when loose object count exceeds a threshold
$ git gc --auto
# Does nothing if thresholds aren't met

# Configure auto-gc threshold (default: 6700 loose objects)
$ git config gc.auto 10000

# Disable auto-gc
$ git config gc.auto 0
```

## Prune Unreachable Objects

```shell
# Prune objects unreachable from any ref
$ git gc --prune=now
# "now" means prune everything — default is 2 weeks grace period

# Just prune (without full gc)
$ git prune
$ git prune --dry-run  # preview what would be pruned

# Default prune expiration
$ git config gc.pruneExpire  # default: 2.weeks.ago
```

## Quiet and Verbose

```shell
# Suppress output
$ git gc --quiet

# Force gc even if another gc seems to be running
$ git gc --force
```

## Reflog Expiration

```shell
# gc also expires old reflog entries
# Default: 90 days for reachable, 30 days for unreachable

# Configure reflog expiration
$ git config gc.reflogExpire 180.days.ago        # reachable entries
$ git config gc.reflogExpireUnreachable 60.days.ago  # unreachable entries

# Expire reflog manually
$ git reflog expire --expire=now --all
$ git gc --prune=now
```

## Check Repository Size

```shell
# Before gc: check repo size and stats
$ git count-objects -v
count: 325              # loose objects
size: 1280              # loose object size in KiB
in-pack: 12500          # packed objects
packs: 3                # number of packfiles
size-pack: 45000        # packfile size in KiB
prune-packable: 100     # loose objects also in packs
garbage: 0
size-garbage: 0

# After gc
$ git gc
$ git count-objects -v
count: 0
size: 0
in-pack: 12825
packs: 1
size-pack: 42000        # better compression
```

## Pack-Related Commands

```shell
# Repack objects (gc does this internally)
$ git repack -a -d
# -a: pack all objects into a single pack
# -d: remove redundant packs

# More aggressive repack
$ git repack -a -d -f --depth=300 --window=300

# Pack refs (branches/tags) into a single file
$ git pack-refs --all
```

## Scheduled Maintenance (Modern Alternative)

```shell
# git maintenance is the modern replacement for manual gc
$ git maintenance run

# Register repo for background maintenance
$ git maintenance register

# Unregister
$ git maintenance unregister

# See also: 42_maintenance.md for full details
```

## Common Workflows

```shell
# After importing a large repo or removing large files from history
$ git gc --aggressive --prune=now

# After force-pushing or deleting many branches
$ git reflog expire --expire=now --all
$ git gc --prune=now

# Regular maintenance
$ git gc  # let it decide what needs doing

# Recover disk space after removing large files with filter-branch/BFG
$ git reflog expire --expire=now --all
$ git gc --prune=now --aggressive

# Check if gc is needed
$ git count-objects -v
# If count is high or size-garbage > 0, run gc
```

## Configuration

```shell
# Auto-gc thresholds
$ git config gc.auto 6700               # trigger at N loose objects (default: 6700)
$ git config gc.autoPackLimit 50         # trigger at N packs (default: 50)

# Prune settings
$ git config gc.pruneExpire "2.weeks.ago"

# Pack settings
$ git config gc.bigPackThreshold 512m    # keep packs larger than this separate
$ git config pack.threads 4              # threads for delta compression
```
