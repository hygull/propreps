# git mv

Moves or renames a file, directory, or symlink and stages the change.

## Basic Usage

```shell
# Rename a file
$ git mv old-name.py new-name.py

# Move a file to a different directory
$ git mv src/utils.py src/helpers/utils.py

# Equivalent to:
$ mv old-name.py new-name.py
$ git rm old-name.py
$ git add new-name.py
```

## Move Multiple Files

```shell
# Move files into a directory
$ git mv file1.py file2.py file3.py src/scripts/

# Move all Python files to a new directory
$ mkdir -p src/legacy
$ git mv src/*.py src/legacy/
```

## Rename with Force

```shell
# Force overwrite if destination exists
$ git mv -f old-file.py existing-file.py

# Without -f, git prevents overwriting:
$ git mv old-file.py existing-file.py
fatal: destination exists, source=old-file.py, destination=existing-file.py
```

## Dry Run

```shell
# Preview without actually moving
$ git mv -n src/utils.py src/helpers/utils.py
Renaming src/utils.py to src/helpers/utils.py

$ git mv --dry-run src/app.py src/main.py
Renaming src/app.py to src/main.py
```

## Move Directories

```shell
# Rename a directory
$ git mv src/old-module src/new-module

# Move a directory inside another
$ git mv src/auth src/modules/auth
```

## Verbose Output

```shell
$ git mv -v src/utils.py src/helpers/utils.py
Renaming src/utils.py to src/helpers/utils.py
```

## Verify Rename Detection

```shell
# After moving, git status shows it as renamed
$ git mv src/utils.py src/helpers/utils.py
$ git status
Changes to be committed:
        renamed:    src/utils.py -> src/helpers/utils.py

# git log follows renames by default with --follow
$ git log --follow src/helpers/utils.py

# See rename detection in diff
$ git diff --staged -M
diff --git a/src/utils.py b/src/helpers/utils.py
similarity index 95%
rename from src/utils.py
rename to src/helpers/utils.py
```

## Case-Sensitive Rename (macOS/Windows)

```shell
# On case-insensitive filesystems, direct rename may not work
$ git mv Readme.md README.md
# If this fails, use a two-step rename:
$ git mv Readme.md temp-readme.md
$ git mv temp-readme.md README.md
```

## Common Workflows

```shell
# Restructure project layout
$ mkdir -p src/{models,views,controllers}
$ git mv models.py src/models/
$ git mv views.py src/views/
$ git mv controllers.py src/controllers/
$ git commit -m "Restructure into MVC layout"

# Rename a module and update imports (manual step needed for imports)
$ git mv src/auth_utils.py src/authentication.py
# Then update all import statements in other files
$ git add -u
$ git commit -m "Rename auth_utils to authentication"
```
