# git rm

Removes files from the working tree and from the staging area (index).

## Basic Usage

```shell
# Remove a file (deletes from disk + stages the deletion)
$ git rm old-file.py
rm 'old-file.py'

# Equivalent to:
$ rm old-file.py
$ git add old-file.py

# Remove multiple files
$ git rm file1.py file2.py file3.py
```

## Keep File on Disk (Remove from Tracking Only)

```shell
# Remove from git tracking but keep the file locally
$ git rm --cached secrets.env
rm 'secrets.env'

# Now add it to .gitignore
$ echo "secrets.env" >> .gitignore
$ git add .gitignore
$ git commit -m "Stop tracking secrets.env"

# Remove an entire directory from tracking
$ git rm --cached -r node_modules/
```

## Remove by Pattern

```shell
# Remove all .log files
$ git rm "*.log"
rm 'error.log'
rm 'access.log'
rm 'debug.log'

# Remove all .pyc files recursively
$ git rm -r "*.pyc"

# Remove all files in a directory
$ git rm -r docs/old/
```

## Force Remove

```shell
# Force remove even if file has staged or unstaged changes
$ git rm -f modified-file.py
rm 'modified-file.py'

# Without -f, git protects you:
$ git rm modified-file.py
error: the following file has local modifications:
    modified-file.py
(use --cached to keep the file, or -f to force removal)
```

## Dry Run

```shell
# Preview what would be removed without actually removing
$ git rm --dry-run "*.log"
rm 'error.log'
rm 'access.log'
rm 'debug.log'

$ git rm -n -r old-directory/
rm 'old-directory/file1.txt'
rm 'old-directory/file2.txt'
```

## Quiet Mode

```shell
# Suppress output
$ git rm -q old-file.py
```

## Common Workflows

```shell
# Scenario: accidentally committed a file that should be ignored
$ git rm --cached .env
$ echo ".env" >> .gitignore
$ git add .gitignore
$ git commit -m "Remove .env from tracking, add to .gitignore"

# Scenario: remove build artifacts from repo
$ git rm -r --cached dist/ build/ __pycache__/
$ git commit -m "Remove build artifacts from tracking"

# Scenario: clean up after renaming (git mv is easier, but manually)
$ mv old-name.py new-name.py
$ git rm old-name.py
$ git add new-name.py
# git detects this as a rename in git status
```

## git rm vs rm

```shell
# rm: only deletes from disk (git still tracks it)
$ rm file.py
$ git status
Changes not staged for commit:
        deleted:    file.py

# git rm: deletes from disk AND stages the deletion
$ git rm file.py
$ git status
Changes to be committed:
        deleted:    file.py
```
