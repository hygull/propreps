# git am

Applies patches from mailbox-formatted files (created by `git format-patch`). Preserves author, date, and commit message.

## Basic Usage

```shell
# Apply a single patch
$ git am 0001-Add-auth-middleware.patch
Applying: Add auth middleware

# Apply multiple patches (in order)
$ git am 0001-Add-auth-middleware.patch 0002-Add-login-endpoint.patch

# Apply all patches in a directory (sorted by filename)
$ git am patches/*.patch
Applying: Add auth middleware
Applying: Add login endpoint
Applying: Add JWT token refresh
```

## Apply from stdin

```shell
# Pipe a combined patch
$ cat all-changes.patch | git am

# Apply from a URL (combine with curl)
$ curl -s https://example.com/fix.patch | git am
```

## Handling Conflicts

```shell
$ git am 0001-Fix-login-bug.patch
Applying: Fix login bug
error: patch failed: src/auth.py:10
Patch failed at 0001 Fix login bug

# Option 1: Fix the conflict and continue
$ vim src/auth.py  # resolve conflicts manually
$ git add src/auth.py
$ git am --continue

# Option 2: Skip this patch
$ git am --skip

# Option 3: Abort entirely (go back to pre-am state)
$ git am --abort

# Option 4: Fall back to 3-way merge (often resolves automatically)
$ git am --3way 0001-Fix-login-bug.patch
$ git am -3 0001-Fix-login-bug.patch
```

## Three-Way Merge (Recommended)

```shell
# Always use 3-way merge for better conflict resolution
$ git am -3 patches/*.patch

# Set as default behavior
$ git config --global am.threeWay true
```

## Preserve Authorship

```shell
# By default, git am preserves original author info
$ git am 0001-Fix-from-colleague.patch
$ git log -1
Author: Colleague <colleague@example.com>  # original author preserved
Commit: You <you@example.com>              # you are the committer
```

## Signoff

```shell
# Add Signed-off-by line to the commit message
$ git am --signoff patches/*.patch
$ git am -s patches/*.patch

# Result:
# Signed-off-by: You <you@example.com>
```

## Interactive Mode

```shell
# Prompt before applying each patch
$ git am -i patches/*.patch
Commit Body is:
--------------------------
Add auth middleware
--------------------------
Apply? [y]es/[n]o/[e]dit/[v]iew patch/[a]ccept all:
```

## Apply from Mailbox

```shell
# Apply patches from an email mailbox file (mbox format)
$ git am < mbox-file

# Apply from a Maildir
$ git am ~/Mail/patches/*
```

## Directory and Exclude

```shell
# Apply patch to a different directory root
$ git am --directory=subproject/ patch.patch

# Exclude files from the patch
$ git am --exclude="*.test.py" patches/*.patch
```

## Common Workflows

```shell
# Receive patches from a mailing list and apply
$ git am -3 -s patches/*.patch

# Apply someone's emailed patch
$ git am -3 /tmp/colleague-fix.patch

# Cherry-pick via patch (alternative to git cherry-pick across repos)
# In repo A:
$ git format-patch -1 abc1234 --stdout > fix.patch
# In repo B:
$ git am -3 fix.patch

# Resume after fixing a conflict
$ git am -3 patches/*.patch
# ... conflict ...
$ vim conflicted-file.py
$ git add conflicted-file.py
$ git am --continue
```

## git am vs git apply

```shell
# git am: creates commits (preserves author, date, message)
$ git am patch.patch      # creates a commit
$ git log -1              # shows the patch as a commit

# git apply: only modifies working tree (no commit)
$ git apply patch.patch   # changes files, nothing staged
$ git status              # shows modified files
```
